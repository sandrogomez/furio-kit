# Dependency Upgrades & Security Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve all 5 currently-known security advisories (1 HIGH, 3 moderate, 1 low) and bring all non-major dependencies to latest. Defer major-version bumps (typescript 6, vite 8) to a follow-up plan because they require code-impact review.

**Architecture:** Phase 1 closes vulnerabilities (Next.js 16.2.3 → 16.2.6 fixes 4 advisories; postcss transitive CVE requires a `pnpm.overrides` because `next@16.2.6` still pins `postcss@8.4.31`). Phase 2 sweeps remaining patch/minor bumps for hygiene. Phase 3 is informational only — major bumps tracked but not executed here.

**Tech Stack:** pnpm, Next.js 16, React 19, TypeScript 5.9, Vite 7, Vitest 4, Biome 2, Tailwind v4.

### Task 4: Force-patch the residual `next → postcss@8.4.31` via `pnpm.overrides`

**Why an override is needed:** `next@16.2.6` still hard-pins `postcss@8.4.31`. Without an override, that copy of postcss remains in the tree and the advisory stays open.

**Files:**
- Modify: `package.json` — add `pnpm.overrides`

- [ ] **Step 1: Add the override**

Edit `package.json` and add (place after `"engines"`):

```json
  "pnpm": {
    "overrides": {
      "postcss": "^8.5.10"
    }
  }
```

- [ ] **Step 2: Re-resolve the lockfile**

```bash
pnpm install
```
Expected: `pnpm-lock.yaml` regenerates; every `postcss` resolves to `^8.5.10`.

- [ ] **Step 3: Verify only one (patched) postcss remains**


Expected: all paths resolve to a single `postcss` version `>= 8.5.10`. Confirm with:

```bash
pnpm audit --json | jq '.advisories | to_entries | map(select(.value.module_name == "postcss")) | length'
```
Expected: `0`.

- [ ] **Step 4: Run the full check matrix**

```bash
rm -rf .next
pnpm lint && pnpm exec tsc --noEmit && pnpm test && pnpm build
```
Expected: all green. Especially watch `pnpm build` — postcss is on the critical CSS path; if anything broke we see it here.

- [ ] **Step 5: Commit**

```bash
git add package.json pnpm-lock.yaml
git commit -m "fix(deps): override postcss to >=8.5.10 (closes GHSA-qx2v-qp2m-jg93)

next@16.2.6 still pins postcss@8.4.31 transitively. pnpm.overrides forces
all postcss installs to the patched line until next upstream releases."
```
